# Tag

**Tag** — тегированный образ (манифест) в репозитории, например `v1.2.0` или `latest`. Это конкретная
версия образа, которую тянет `docker pull`. Tag закрывает задачу «увидеть версии образа, их digest,
размер, платформу и когда их последний раз скачивали» — без ручного обхода OCI-манифестов.

Tag — **read-only проекция из хранилища** (source of truth — zot); он не хранится в БД
kacho-registry. Тег появляется при `docker push <repo>:<tag>` и физически удаляется единственным
control-plane-путём — `DeleteTag` (data-plane `DELETE` манифеста отвергается — образы удаляются только
через control-plane, чтобы удаление проходило authz-проверку и корректно снимало repo-tuple при
опустошении репозитория).

## Методы

Теги читаются проекцией; единственная мутация control-plane — `DeleteTag` (async).

### ListTags — список тегов репозитория

<ApiOperation method="GET" endpoint="/registry/v1/registries/{registryId}/repositories/{repository}/tags">

Возвращает теги репозитория (проекция из zot) с cursor-пагинацией. RPC **scope-filtered**: per-repo
Check `v_list` на `registry_repository` выполняется в хендлере; нет доступа → `NOT_FOUND`
(existence-hiding).

<CodeBlock language="bash">
  {dedent`
    curl 'http://localhost:18080/registry/v1/registries/reg7h9x2k4m8p0q1r5/repositories/backend/api/tags?pageSize=50' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

<CodeBlock language="json">
  {dedent`
    {
      "tags": [
        {
          "registryId": "reg7h9x2k4m8p0q1r5",
          "repository": "backend/api",
          "tag": "v1.2.0",
          "digest": "sha256:9b2c…e41",
          "sizeBytes": "13107200",
          "mediaType": "application/vnd.oci.image.manifest.v1+json",
          "createdAt": "2026-07-10T10:04:00Z",
          "architecture": "linux/amd64",
          "lastPulledAt": "2026-07-10T11:20:00Z",
          "pushedBy": "sva7k2…",
          "downloadCount": "12"
        }
      ],
      "nextPageToken": ""
    }
  `}
</CodeBlock>

</ApiOperation>

### DeleteTag — удалить тег

<ApiOperation method="DELETE" endpoint="/registry/v1/registries/{registryId}/repositories/{repository}/tags/{tag}" async>

Удаляет тег из репозитория — единственный destructive-путь для образов (data-plane `DELETE` манифеста
отвергается `405`). Per-repo Check `v_delete` на `registry_repository` выполняется синхронно в
хендлере **до** создания операции. Удаление последнего тега опустошает репозиторий и снимает его
repo-tuple. Ответ — `Operation` (см. [Операции](/api/operations)).

<CodeBlock language="bash">
  {dedent`
    curl -X DELETE 'http://localhost:18080/registry/v1/registries/reg7h9x2k4m8p0q1r5/repositories/backend/api/tags/v1.2.0' \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

</ApiOperation>

## Ошибки и граничные случаи

<table>
  <thead><tr><th>Ситуация</th><th>Код / поведение</th></tr></thead>
  <tbody>
    <tr><td>Malformed <code>registryId</code></td><td><code>INVALID_ARGUMENT</code> (<code>"invalid registry id '&lt;X&gt;'"</code>)</td></tr>
    <tr><td>Нет <code>v_list</code> / <code>v_delete</code> на репозиторий</td><td><code>NOT_FOUND</code> (existence-hiding, не 403)</td></tr>
    <tr><td>Удаление несуществующего тега</td><td><code>NOT_FOUND</code></td></tr>
    <tr><td>Попытка удалить образ через data-plane <code>DELETE</code></td><td><code>405</code> (удаление только через <code>DeleteTag</code>)</td></tr>
    <tr><td>zot недоступен</td><td><code>UNAVAILABLE</code> (fail-closed)</td></tr>
  </tbody>
</table>

:::tip Освобождение имени реестра
Реестр удаляется только пустым. Удалите все теги во всех репозиториях реестра через `DeleteTag`; когда
namespace пуст, [Registry.Delete](/api/registry) проходит (иначе — `"registry is not empty"`).
:::
